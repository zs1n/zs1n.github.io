Veo que una contrase√±a `gpp` para el usuario `svc_tgs` y que esta encriptada, se que esta en ese formato ya que me lo indica el nombre del atributo `cpassword` y ademas el archivo estaba guardado en la ruta `/Preferences` que es ahi donde las `_Group Policy Preferences_ (GPP)` guardaban configuraciones dentro de las `GPOs`. asi que voy a usar `gpp-decrypt` para romper el hash.

```bash
gpp-decrypt edBSHOwhZLTjt/QS9FeIcJ83mjWA98gw9guKOhJOdcqh+ZGMeXOsQbCpZ3xUjTLfCuNH8pG5aSVYdYw/NglVmQ

```

Ahora tengo las credenciales `GPPstillStandingStrong2k18` del usuario `svc_tgs`, ahora con este usuario voy a performar un ataque `Kerberoasting` para ver si puedo recuperar hashes `kerberos` de algun usuario del dominio y sino me toca enumerar el dominio desde adentro logueandome con `smbclient`.
